# Contributing to QuantumCSS

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to QuantumCSS. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see and why.

### Suggesting Enhancements
- Explain why this enhancement would be useful to most QuantumCSS users.
- Provide examples of how the feature would be used.

### Pull Requests
- Follow the existing code style.
- Ensure all tests pass (`npm test`).
- Document any new features or changes in the `docs/` folder.
- Update the `CHANGELOG.md` with your changes.

## Development Setup

1. Fork the repo and clone it locally.
2. Install dependencies: `npm install`.
3. Create a branch for your work.
4. Make your changes and add tests if applicable.
5. Submit a PR!

Thank you for your contributions!
